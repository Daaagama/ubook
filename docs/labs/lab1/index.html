Опис предметної області

Вступ

Предметною областю є інтернет-магазин електронних книг, зокрема його створення та забезпечення зручності використання. Далі буде описано вимоги до WEB-додатку який розроблюватиметься.

Основні визначення

Електронна комерція — це сфера цифрової економіки, що включає всі фінансові та торгові транзакції, які проводяться за допомогою комп'ютерних мереж, та бізнес-процеси, пов'язані з проведенням цих транзакцій. До електронної комерції відносять електронний обмін інформацією (англ. Electronis Data Interchange, EDI), електронний рух капіталу (англ. Electronic Funds Transfer, EFT), електронну торгівлю (англ. E-Trade), електронні гроші (E-Cash), електронний маркетинг (англ. E-Marketing), електронний банкінг (англ. E-Banking), електронні страхові послуги (англ. E-Insurance) тощо.[1]

Інтернет-магазин — сайт, з якого можна вибрати та замовити потрібний товар чи послугу.[2]

Електронна книга — версія книги в електронному вигляді, яку можна читати за допомогою комп'ютерів, мобільних телефонів чи спеціалізованих пристроїв. Електронні книги можуть розповсюджуватися у файлах різних форматів, наприклад:

  • простий текст;
  • текстові файли з оформленням (HTML, OPF FlipBook, OpenDocument, SGML, XML, FictionBook (.fb2), TeX, PDF, HTMLHelp (.chm), Microsoft (.lit), eReader, PostScript (.ps, .eps), ExeBook, Mobipocket (.prc) та ін.);
  • графічні растрові (TIFF, JPEG, DjVu тощо);
  • мультимедіа книги (SWF, EXE та ін.).[3]

Вимоги до продукту

Важливими елементами інтернет-магазину є:

  • оновлення наявного асортименту
  • можливість додавати товари до кошика
  • вхід для зареєстрованих користувачів
  • можливість пошуку товарів
  • безпека користувацьких даних

Розгланемо кожен пункт

  1. Оновлення наявного асортименту ubook міститиме можливість додавання адміністратором сайту нових книг та оновлення інформації про вже додані товари.
  2. Можливість додавати товари до кошика Кошик інтернет-магазину це сторінка до якої користувач надсилає товари, щоб потім їх придбати. Функції кошику:
    • перевірити які товари було обрано та завершити оформлення замовлення
    • дізнатися скільки коштує покупка
    • перевірити чи всі необхідні товари обрано
      • Отже, розроблюваний продукт матиме зручний кошик, де зручно переглядати всі обрані товари. Надаватиме можливість повернутися до покупок, щоб додати товар. Відображатиме ціну окремих обраних товарів, а також загальну ціну. Надаватиме можливість редагувати кошик, наприклад щоб видаляти з нього небажаний товар.
  3. Вхід для зареєстрованих користувачів ubook надаватиме можливість створення облікового запису, для більш зручного користування магазином. У користувачів, які увійшли в обліковий запис, буде можливість зберігати товари в список бажань. А також можна буде зберігати інформацію потрібну для покупки, щоб пришвидшити оформлення замовлення.
  4. Можливість пошуку товарів Головна задача пошуку - допомогти користувачу знайти потрібний товар. Тому від реалізації функціональності пошуку залежить зручність користування. Головні вимоги до пошуку в інтернет-магазині:
    • простота та швидкість знаходження товару покупцем
    • сприяння скороченню кількості дій для покупки потрібного товару
    • зручність при здійсненні покупки
    Тож розроблюється продукт котрий буде задовільняти ці вимоги.
  5. Безпека користувацьких даних Для безпечного використання сайту передбачені надійні протоколи — https, та інші системи безпеки.

Юзкейси

Користувач

  • Пошук по каталогу книг

    Користувач має можливість шукати книги, застосовувати фільтри до результату пошуку, сортувати результати за певним критерієм.

  • Перегляд опису книг

    Користувач може переглядати детальну інформацію про книгу на її сторінці. Також користувач має можливість читати коментарі інших користувачів.

  • Оформлення замовлення

    Користувач може оформити замовлення, додавши попередньо книги до корзини. Користувач повинен вказати потрібні для покупки дані та може перевірити обрані товари. Після підтвердження замовлення з'являється у адміністратора.

  • Написання відгука

    Користувач може залишити відгук до будь-якої книги, перейшовши на її сторінку. Всі написані користувачами коментарі будуть відображатися на сторінці книги.

Адміністратор

  • Менеджмент книг

    Адміністратор може додавати нові книги, редагувати інформацію про книги та видаляти книги зі списку книг.

  • Менеджмент замовлень

    Адміністратор може управляти замовленнями користувачів. Оброблювати та підтверджувати їх.
